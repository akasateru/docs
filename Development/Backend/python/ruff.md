## 1. ruff概要

- Pythonのコードフォーマッターは、2026年現在 **「Ruff」の一強状態**
- これまで「Black（整形）」「isort（インポート順）」「flake8（ミス指摘）」と別々に導入していたツールが、Ruff1つで済む
- Black ほど柔軟にコード全体を整形できるわけではないため、完全なフォーマットを求める場合は Black と併用することが一般的

### 1.1. 特徴

1. 高速
2. 多様なLinter
3. 設定が簡単
   1. pyproject.tomlや.ruff.toml などで細かい設定ができる

## 2. ruff以外の選択肢

| ツール名    | 特徴                                       | 思想                                                       |
| ----------- | ------------------------------------------ | ---------------------------------------------------------- |
| Ruff (推奨) | 圧倒的な爆速。 Rust製。                    | Black互換 + インポート整列 + リンター。これ1つで完結。     |
| Black       | 現代の標準。「妥協のない」フォーマッター。 | 設定を最小限にし、誰が書いても同じコードにする。           |
| autopep8    | PEP 8準拠に特化。                          | 既存のコードスタイルを大きく崩さず、規約違反だけ直す。     |
| YAPF        | Google製。柔軟なカスタマイズ。             | 読みやすさを追求。設定次第でかなり細かく挙動を変えられる。 |

## 3. 使い方

### 3.1. インストール方法

```bash
pip install ruff
```

### 3.2. 基本コマンド

コードスタイルチェック

```bash
ruff check .

```

コード自動修正

```bash
ruff check . --fix
```

### 3.3. 設定ファイル

```toml
[tool.ruff]
select = ["E", "F", "I"]
ignore = ["E501"]
max-line-length = 120
```
