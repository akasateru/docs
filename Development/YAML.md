## 1. YAML

### 1.1. 使用用途

- **設定ファイル（Configuration Files）**
    最も一般的な用途です。Docker, Kubernetes, CI/CDツール（GitLab CI, CircleCI, GitHub Actions）など、モダンなツールの設定に多用されます。
- **データ交換フォーマット（Data Interchange Format）**
    プログラミング言語間や、サービス間でデータをやり取りする際に使用されます。
- **データ構造化の記述フォーマット**
    JSONやXMLと同様に、階層構造を持つデータを記述するために用いられます。
- **IaC（Infrastructure as Code）定義ファイル**
    インフラの構成をコードで管理するツールにおいて、設定を記述する形式として採用されています。
