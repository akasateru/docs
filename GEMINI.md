# 1. Gemini CLI プロジェクト要件 (Project Mandates)

このファイルは、`docs` ディレクトリおよびそのサブディレクトリ内で Gemini CLI が動作する際の基本的な要件と運用ガイドラインを定義します。これらの指示は、一般的なデフォルト設定よりも絶対的に優先されます。

## 1.1. ドキュメントの標準化

- **言語:** 既存のファイル内容やユーザーからの指定がない限り、ドキュメントは日本語で記述してください。ドキュメントは体言止めで作成してください。
- **フォーマット:** GitHub Flavored Markdown を使用してください。見出し、リスト、コードブロックなどを一貫したスタイルで記述してください。
- **構成:** 既存のディレクトリ構造（例: `Cloud/GCP`、`Web/Frontend` など）を維持してください。新しいノートは最も関連性の高いカテゴリに配置してください。

## 1.2. ドキュメントにおけるエンジニアリング基準

- **コンテキストの優先:** コンテキストとスタイルの一貫性を保つため、ファイルを更新する前に必ず既存の内容を読み取ってください。
- **局所的な更新:** 既存のノートへの不要な変更を最小限に抑えるため、更新には対象を絞った置換（`replace` ツール）を使用してください。
- **相互参照:** 他のドキュメントに関連するコンテンツを追加する場合は、適切な箇所に内部のマークダウンリンクを含めてください。

## 1.3. 運用ガイドライン

- **簡潔さ:** 技術的な要約は直接的でノイズの少ない（要点を絞った）ものにしてください。不要な装飾や冗長な説明は避けてください。
- **検証:** ファイルの更新後、Markdownの構造が適切であること、内容がユーザーの意図と一致していることを確認してください。
