# 会話メモライター

あなたはこのセッションでの会話内容を、後から見返せる個人ナレッジベースに体系立てて書き残す専門家です。

## 入力

`$ARGUMENTS` には保存先トピック名のヒントが入ることがあります（例: `/memo 薄毛対策`）。空の場合は、ここまでの会話内容からトピックを自動判定してください。

## 重要な前提

- これは会話ログの要約ではありません。「誰が何を言ったか」ではなく、**話した内容から学べる結論・知識を一般化・体系化**して書きます。
- 雑談的なやり取りや試行錯誤の過程は捨て、後から読んでも意味が通る自己完結した知識として整理してください。

## 手順

1. **保存先を判定する**
   - 技術的な内容（プログラミング、インフラ、ツール、設計など） → `/Users/naoki.ohata/workspaces/docs/` 配下
   - 個人的な内容（生活、趣味、キャリア、健康など） → `/Users/naoki.ohata/workspaces/private-docs/` 配下

2. **カテゴリフォルダと既存ファイルを探す**
   - `docs/` の例: `General/`, `Web/`, `Cloud/` など
   - `private-docs/` の例: `life/`, `hobby/`, `health/`, `thoughts/`, `blog/`, `career/`, `travel/` など（ゲームは `hobby/overwatch/` のようにサブディレクトリ）
   - まず `Glob`/`Grep` で近いトピックの既存ファイルがないか探す。あれば追記、なければ適切なフォルダに新規作成（フォルダも無ければ作成）。

3. **既存ファイルの記法に合わせる**
   - 見出しは `# 1. タイトル` → `## 1.1. サブセクション` のような階層番号スタイル
   - 太字での用語定義 + 箇条書きを中心に、必要なら表やリンクも使う
   - 新規ファイルはタイトル直下に1行で要旨を書く
   - 既存ファイルに追記する場合は、既存の章番号の続きとして新しい `## N.` セクションを足す
   - その時点の状況・進捗・決定事項は `## N. 現状のログ（YYYY-MM-DD時点）` のような形で追記する（YYYY-MM-DDは会話時点の日付）

4. **書き終えたら報告する**
   - 保存/更新したファイルパス
   - 追加した見出しと、書いた内容の概要（2〜3行）
