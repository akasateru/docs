# 1. WAF（Web Application Firewall）

## 1.1. 概要

WAFはWebアプリケーション層（L7）に対する攻撃を検知・遮断するファイアウォール。ネットワーク層のFWでは防げないSQLインジェクションやXSSなど、HTTPリクエストの中身を見て判断する必要がある攻撃に対応する。

## 1.2. 一般的なFWとの違い

| 項目 | 一般的なFW（L3/L4） | WAF（L7） |
| ---------- | ------------------------------ | ---------------------------------- |
| 監視対象 | IPアドレス・ポート番号 | HTTPリクエストの中身（パラメータ、ヘッダ、Body） |
| 防げる攻撃 | 不正アクセス、ポートスキャンなど | SQLインジェクション、XSS、CSRFなど |

## 1.3. 検知方式

- **シグネチャ型**: 既知の攻撃パターン（正規表現やルール）に一致するリクエストをブロック。誤検知は少ないが未知の攻撃には弱い。
- **異常検知型**: 通常のトラフィックからの逸脱をスコアリングして判定。未知の攻撃にも対応できるが誤検知（False Positive）が増えやすい。

## 1.4. 主なサービス例

- クラウド型: AWS WAF、Cloudflare WAF、Google Cloud Armor
- アプライアンス型: F5, Imperva など

## 1.5. 運用時の注意点

- 導入初期はブロックモードではなく検知（ログのみ）モードで運用し、誤検知でユーザーの正常な操作をブロックしていないか確認してから本番遮断に切り替える。
- ルールセットは定期的に更新し、OWASP Top 10などの最新の脅威に追従する。
- WAFだけに依存せず、アプリケーション側でも入力値検証・エスケープ処理を行う多層防御が基本。
