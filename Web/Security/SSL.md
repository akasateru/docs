# 1. SSL/TLS

## 1.1. 概要

SSL/TLSは通信の暗号化・改ざん検知・通信相手の認証を行うプロトコル。現在主に使われているのはTLS（SSLは旧称・脆弱性のため非推奨）だが、慣習的に「SSL証明書」のように呼ばれ続けている。

## 1.2. 仕組みの概略

1. クライアントがサーバーに接続要求（ClientHello）。
2. サーバーが証明書（公開鍵を含む）を提示。
3. クライアントが証明書をCA（認証局）のルート証明書と照合して検証。
4. 共通鍵（セッションキー）を安全に交換（鍵交換）。
5. 以降の通信は共通鍵暗号方式で高速に暗号化される。

公開鍵暗号は鍵交換にのみ使い、実際のデータ通信は計算コストの低い共通鍵暗号を使うハイブリッド方式。

## 1.3. 証明書の種類

| 種類 | 検証レベル | 用途 |
| -------------------------------- | ----------------------------------- | ------------------------------ |
| DV（Domain Validation） | ドメイン所有権のみ確認 | 個人サイト、社内サービスなど |
| OV（Organization Validation） | 組織の実在性も確認 | 企業サイト |
| EV（Extended Validation） | 最も厳格な審査 | 金融機関など高い信頼性が必要な場合 |

## 1.4. よく使う確認コマンド

```bash
# 証明書の有効期限や発行者を確認
openssl s_client -connect example.com:443 -servername example.com | openssl x509 -noout -dates -issuer
```

## 1.5. 運用上の注意点

- 証明書の有効期限切れはサイト全体の閲覧不可に直結するため、自動更新（Let's EncryptのACMEなど）を基本にする。
- 中間証明書（Intermediate CA）が欠けていると、一部ブラウザ・環境でのみ警告が出ることがある（フルチェーンでの提供が必須）。
- TLS 1.0/1.1は非推奨。TLS 1.2以上をサーバー側で強制する。
