## 1. 基本構成

| パーツ名             | 役割（例え）   | 説明                                                                         |
| -------------------- | -------------- | ---------------------------------------------------------------------------- |
| CPU                  | 頭脳           | データの処理速度を決めます。IntelのCore iやAMDのRyzenが有名です。            |
| メモリ (RAM)         | 机の広さ       | 作業中データを一時保存する場所。広いほど同時にたくさんのアプリを動かせます。 |
| ストレージ (SSD/HDD) | 引き出し・倉庫 | 写真や動画、アプリを保存する場所。現在は読み書きが速いSSDが主流です。        |
| GPU (グラボ)         | 絵師           | 映像の処理に特化した頭脳。3Dゲームや動画編集には欠かせません。               |
| OS                   | 言語・人格     | WindowsやmacOSなど、PCを動かすための基本ソフトです。                         |

## 2. プロセッサ(CPU)

- プロセッサ: 2.4GHz 8コア Intel Core i9

### 2.1. コア数ごとの作業イメージ

| コア数         | 開発での体感レベル | 具体的な動作のイメージ                                                       |
| -------------- | ------------------ | ---------------------------------------------------------------------------- |
| 4コア          | 最低限             | エディタとブラウザだけなら動くが、Dockerを立ち上げると全体が重くなる。       |
| 6〜8コア       | 標準・快適         | ブラウザで調べ物をしながら、バックグラウンドでサーバーを動かしてもサクサク。 |
| 10〜12コア以上 | プロ仕様           | 重いアプリのコンパイル（ビルド）が爆速。複数の仮想環境を立てても余裕。       |

### 2.2. なぜWeb開発で「多コア」が重要なのか？

Web開発の現場では、目に見えないところで多くの「作業員（スレッド）」が働いています。

エディタの入力補完: コードを書いている最中に、ミスがないか裏で常にチェックが走ります。

ローカルサーバー: 保存した瞬間にプログラムを再起動し、ブラウザに反映させます。

仮想環境 (Dockerなど): PCの中にもう一台のPC（Linux）を動かすようなものなので、専用のコアを割り当てる必要があります。

通信・会議アプリ: ZoomやSlackのビデオ通話は、意外とCPUコアを占有する。
