## 3. 評価

### 3.1. 混同行列（Confusion Matrix）

- **TP (True Positive)**: 陽性と予測し、正解も陽性（的中）
- **FP (False Positive)**: 陽性と予測したが、正解は陰性（空振り）
- **FN (False Negative)**: 陰性と予測したが、正解は陽性（見逃し）
- **TN (True Negative)**: 陰性と予測し、正解も陰性（的中）

### 3.2. 各指標の定義と意味

#### 3.2.1. 精度（Precision / 適合率）

「陽性と予測したもの」のうち、実際に陽性だった割合。「無実の人を犯人扱いしたくない（誤検出を減らしたい）」ときに重視。

Precision\=TP+FPTP​

- **例**: スパムメール判定。普通のメールをスパムと誤判定すると困るため、精度が重要。

### 3.3. 再現率（Recall / 網羅率）

「本当に陽性のもの」のうち、正しく陽性と予測できた割合。「見逃しを絶対に避けたい」ときに重視。

Recall\=TP+FNTP​

- **例**: がん診断や火災検知。見逃すと重大な被害が出るため、再現率が重要。

### 3.4. F1スコア（F1-score）

精度と再現率の**調和平均**をとった指標。 片方の数値だけが高く、もう片方が極端に低い場合に値が下がるようになる。モデルの全体的なバランスを評価するのに適している。

F1\=2⋅Precision+RecallPrecision⋅Recall​

## 4. 使い分けの基準

| 指標                 | 重視するシーン             | 特徴                               |
| -------------------- | -------------------------- | ---------------------------------- |
| **精度 (Precision)** | 誤検出（FP）を避けたいとき | 「当たった時の信頼度」が高い       |
| **再現率 (Recall)**  | 見逃し（FN）を避けたいとき | 「漏れなく拾う力」が高い           |
| **F1スコア**         | 両方のバランスを見たいとき | どちらかが極端に低いと数値が下がる |
