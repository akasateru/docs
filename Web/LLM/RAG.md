# RAG

## 1. 概要

RAG（Retrieval-Augmented Generation：検索拡張生成）は、大規模言語モデル（LLM）が回答を生成する前に、社内文書やデータベースなどの外部情報を検索し、その情報を元に回答を作成する技術。

### 1.1. RAGのメリット

- **高精度・高信頼性:** 正確なソース（外部情報）に基づくため、事実に基づいた回答が可能。
- **ハルシネーションの低減:** モデルの知識だけでなく、提示された資料から回答するため虚偽情報が減る。
- **最新・固有情報の活用:** 社内規定、マニュアル、最新のWeb情報などを利用できる。
- **コスト削減:** 再学習（ファインチューニング）より低コストで独自のAIを構築できる。

### 1.2. 主な活用事例

- **社内FAQ・ヘルプデスク:** 社内規程や過去の知見を検索し、回答を作成。
- **文書要約・分析:** 長大な報告書やマニュアルの特定箇所を抽出して要約。
- **カスタマーサポート:** 製品マニュアルを参照して、正確なサポートを行う

### 1.3. RAGの技術的要素

- **ベクトルデータベース:** テキストを数値（ベクトル）化して高速に検索する仕組み（例: Pinecone, Milvus）。
- **Embeddingモデル:** テキストをベクトルに変換するモデル。
- **LLM:** 生成を担当するモデル（例: GPT-4, Claude, Llama）

## 2. 仕組み

- **検索 (Retrieval):** ユーザーの質問に関連する情報をデータベースから抽出。
- **拡張 (Augmented):** 抽出した情報（コンテキスト）をユーザーの質問に付与。
- **生成 (Generation):** LLMが付与された情報を踏まえて回答を作成。

## 3. 例

FAQの場合
1. ユーザーの質問をベクトル化　→ 検索クエリベクトル
2. FAQの各文章もベクトル化 → 文章ベクトルを事前に保存
3. 類似度計算 → コサイン類似度などで比較
4. 上位N件取得 → スコアが高い順に取得
5. コンテキスト作成 → 検索結果を結合し「情報源」として準備
6. LLMに渡す → 質問+情報源を一緒にプロンプトに入れて、LLMが回答生成
