## 1. ディレクトリ構成

情報整理においてディレクトリ構成は重要
必要な情報を見つけやすいのに加え、情報の整理に役立つ

```bash
my-product-docs/
├── 01_Concepts/            # 【概念】プロダクトを「点」で理解する
│   ├── business-logic/     # 業界知識、計算ロジック、制約事項
│   └── data-models/        # ER図、エンティティの定義
├── 02_Solutions/           # 【ガイド】プロダクトを「線」で理解する
│   ├── use-cases/          # 「注文する」「退会する」などの主要ユースケース
│   └── user-flows/         # 画面をまたぐ一連の動作（既存の機能・画面を繋ぐ）
├── 03_Reference/           # 【リファレンス】プロダクトを「詳細」に記録する
│   ├── screens/            # 画面仕様（コンフルの「画面」に対応）
│   └── features/           # 機能仕様（コンフルの「機能」に対応）
└── 04_Resources/　         # リンク、画像、議事録など
```
