## 1. Java

- 「一度書けば、どこでも動く（Write Once, Run Anywhere）」という理念のもと設計されている。

### 1.1. Javaの主な特徴

- プラットフォームに依存しない（JVM）
  - Javaで書かれたプログラムは、**Java仮想マシン（JVM）**の上で動作。OS（Windows, Mac, Linuxなど）に依存せず、同じコードを動かすことができる。
- オブジェクト指向
  - 「モノ（オブジェクト）」を組み立てるようにプログラムを構築する手法。コードの再利用性が高く、大規模な開発に適している。
- 堅牢性とセキュリティ
  - メモリ管理（ガベージコレクション）を自動で行うため、メモリ漏れなどのバグが発生しにくく、セキュリティ機能も強力。
- 豊富なライブラリとコミュニティ
  - 長い歴史があるため、開発に役立つ部品（ライブラリ）が非常に充実しており、トラブルの際もネット上に解決策が多く見つかる。

### 1.2. Javaプログラムが動く仕組み

- Javaは「コンパイル」と「実行」の2段階を経て動作。
  - ソースコード (.java): 人間が書くコード。
  - バイトコード (.class): コンパイラがJVM用に変換した中間コード。
- JVMによる実行: 各OSのJVMが、そのコンピュータが理解できる形式に翻訳して実行。

## 2. 基本構文

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, Java!"); // コンソールに文字を表示
    }
}
```

- クラス（class）: プログラムの枠組み。
- メソッド（method）: 実際の処理（手続き）を書く場所。
