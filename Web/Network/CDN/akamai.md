## 2. Akamaiの主要サービス

### 2.1. Content Delivery (CDN)

最も有名な機能。画像、動画、HTML などをユーザーのすぐ近くのサーバーにキャッシュし、高速配信する。

- **Google Cloud との比較:** Google Cloud の CDN よりも拠点が圧倒的に多く、世界のどこからでも低レイテンシで配信できるのが強み。

### 2.2. Edge DNS

世界中に分散された権威 DNS サービス。

- **特徴:** 100% の稼働率（SLA）を保証しており、大規模な DDoS 攻撃を受けても DNS 応答が止まりません。ドメイン解決（名前解決）の段階で、最も近い Edge サーバーへ誘導する役割も担っています。

### 2.3. EdgeWorkers (エッジコンピューティング)

Cloud Run よりもさらにユーザーに近い場所で、**JavaScript コードを実行**できる FaaS（Function as a Service）です。

- **活用例:**

  - ユーザーの国やデバイスに応じて HTTP ヘッダーを書き換える。
  - A/B テストの振り分けをエッジで行い、オリジン（Cloud Run 等）の負荷を減らす。
  - セキュリティチェックをエッジで完結させる。

## 3. エンジニアが Akamai を導入する主な理由

自社プロダクトを Cloud Run や GCE で動かしている場合でも、その「手前」に Akamai を置く構成はよくあります。

| 目的               | 内容                                                                                                         |
| ------------------ | ------------------------------------------------------------------------------------------------------------ |
| **DDoS 対策**      | 攻撃がクラウド（GCP）に届く前の「エッジ」で遮断するため、バックエンドがパンクしない                          |
| **グローバル対応** | 海外ユーザーが多い場合、東京リージョンの Cloud Run だけでは遅いため、Akamai で世界中にキャッシュをばら撒く。 |
| **負荷分散**       | 複雑なルーティング（特定のパスだけ別のサーバーに飛ばす等）を、エッジ側で柔軟に制御できる。                   |
