## 1. SSH(Secure Shell)

SSH（Secure Shell）は、暗号化と認証技術を用いてネットワーク経由で遠隔地のコンピュータ（サーバ）を安全に操作・管理するための通信プロトコル

## 2. sshコマンド

### 2.1. 良く使うオプション

| オプション | 用途                             | コマンド例                       |
| ---------- | -------------------------------- | -------------------------------- |
| **-p**     | ポート番号を指定                 | `ssh -p 2222 user@host`          |
| **-i**     | 秘密鍵ファイルを指定             | `ssh -i ~/.ssh/id_rsa user@host` |
| **-l**     | ログインユーザーを指定           | `ssh -l user host`               |
| **-v**     | デバッグモード（接続状況を表示） | `ssh -v user@host`               |
| -T         |

### 2.2. 関連コマンド

- **`ssh-keygen`**: 公開鍵と秘密鍵のペアを作成する。
- **`ssh-copy-id`**: 公開鍵をリモートサーバーに登録する（パスワードなしログイン用）。
- **`scp`**: SSHを使ってファイルを安全に転送する。
- **`sftp`**: 安全なファイル転送プロトコル（FTPの安全版）。

### 2.3. セキュリティと認証

SSHの認証は大きく2種類。

1. **パスワード認証**: アカウントのパスワードを使用。
2. **公開鍵認証**: 鍵ペア（秘密鍵・公開鍵）を使用。セキュリティが高く、自動化に適している。

## 3. ~/.ssh/config

[\~/.ssh/configについて #SSH - Qiita](https://qiita.com/passol78/items/2ad123e39efeb1a5286b)

.ssh/configは、ssh経由でのリモートサーバーの接続する際に利用される設定ファイル

一般的な書き方の問題点は、指定が増えるとコマンドが複雑になる

```bash
# 一般的な接続方法
ssh hoge.example.com

# ユーザー名を指定する場合は以下
ssh ユーザー名@hoge.example.com

# ポート名を指定する場合
ssh ユーザー名@ホスト名 -p ポート番号

# 公開鍵認証の場合
ssh ユーザー名@ホスト名 -i ~/.ssh/鍵の名前

# ポート名及び、公開鍵認証の場合
ssh ユーザー名@ホスト名 -i ~/.ssh/鍵の名前 -p ポート番号
```

.ssh/configでは以下のように書ける

```bash
Host 任意の接続名(hoge)
HostName ホスト名
User ユーザー名
Port ポート番号
IdentityFile 鍵へのPATH(例えば~/.ssh/hoge.key)
```

このように設定を書いておくと`ssh hoge`のように簡単に接続できる
