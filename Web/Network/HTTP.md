# 1. HTTP (HyperText Transfer Protocol)

HTTPは、WebブラウザとWebサーバ間でデータを送受信するための通信プロトコルです。

## 1.1. HTTPの基本概念

- **リクエスト/レスポンス**: クライアント（ブラウザ）がリクエストを送り、サーバがレスポンスを返すステートレスなプロトコル。
- **メソッド**:
  - `GET`: リソースの取得
  - `POST`: データの送信・リソースの作成
  - `PUT`: リソースの更新（全体）
  - `PATCH`: リソースの部分更新
  - `DELETE`: リソースの削除
- **ステートレス性**: 各リクエストは独立しており、サーバは過去のリクエスト状態を保持しない（Cookieやセッションで補完）。

## 1.2. HTTPステータスコード

主なコード体系：

- **2xx (Success)**: リクエスト成功（例: `200 OK`, `201 Created`）
- **3xx (Redirection)**: 追加の操作が必要（例: `301 Moved Permanently`, `302 Found`）
- **4xx (Client Error)**: クライアント側のエラー（例: `400 Bad Request`, `401 Unauthorized`, `403 Forbidden`, `404 Not Found`）
- **5xx (Server Error)**: サーバ側のエラー（例: `500 Internal Server Error`, `503 Service Unavailable`）

## 1.4. クロスサイトリクエストとプライバシー保護 (ITP)

現代のWebブラウザでは、ユーザーのプライバシー保護を目的に、ドメインを跨ぐ（クロスサイト）リクエストやトラッキングに対する制限が強化されています。

### 1.4.1. ITP (Intelligent Tracking Prevention)

AppleのSafariに搭載されているトラッキング防止機能です。

- **目的**: ユーザーの行動を複数のサイトにわたって追跡する「クロスサイトトラッキング」を制限する。
- **主な仕組み**:
  - **サードパーティCookieのブロック**: 訪問しているサイト以外のドメインが発行するCookieを全面的にブロックします。
  - **ファーストパーティCookieの制限**: JavaScript（`document.cookie`）で生成されたCookieの有効期限を、特定条件下で短期間（24時間〜7日間など）に短縮します。
  - **CNAMEクローキング対策**: 広告配信ドメインを自社サブドメインのように見せかける手法を検知し、制限をかけます。
- **影響**:
  - アフィリエイト広告のコンバージョン計測
  - アクセス解析（Google Analyticsなど）でのユーザー同一性保持
  - リターゲティング広告の精度低下

### 1.4.2. その他の制限

- **Google Chrome (Privacy Sandbox)**: サードパーティCookieの段階的廃止と、プライバシーに配慮した代替技術の導入。
- **SameSite属性**: Cookieに`SameSite=Lax`や`Strict`を設定することで、CSRF（クロスサイトリクエストフォージェリ）対策や意図しないクロスサイト送信を制御。
