# 1. Auth0

## 1.1. 概要

Auth0は認証・認可をSaaSとして提供するIDaaS（Identity as a Service）。ログイン画面、ソーシャルログイン、MFA、セッション管理などを自前実装せずに導入できる。

## 1.2. 主な機能

- **Universal Login**: Auth0がホストするログイン画面を組み込める。自前でパスワード管理をしなくて良い。
- **ソーシャルログイン**: Google、GitHubなど各種IdP連携をダッシュボードから有効化するだけで使える。
- **MFA（多要素認証）**: SMS、TOTP、生体認証などをポリシーベースで設定可能。
- **Rules / Actions**: ログイン時のフローにカスタムロジック（ロール付与、外部API連携など）を差し込める。

## 1.3. 認証フロー（Authorization Code Flow）の概略

1. アプリがAuth0のログイン画面へリダイレクト。
2. ユーザーが認証情報を入力しログイン。
3. Auth0が認可コードをアプリのコールバックURLに返す。
4. アプリがバックエンドで認可コードをアクセストークン・IDトークンと交換。
5. 取得したトークンでAPIへのアクセスやユーザー情報の取得を行う。

SPAやモバイルアプリではPKCE（Proof Key for Code Exchange）を組み合わせて、認可コード横取り攻撃を防ぐのが標準。

## 1.4. トークンの種類

| トークン       | 役割                                   |
| -------------- | ---------------------------------------- |
| ID Token       | ユーザーが誰であるかを示す（OpenID Connect） |
| Access Token   | APIへのアクセス権限を示す（OAuth 2.0）  |
| Refresh Token  | Access Tokenの再発行に使う長期トークン  |

## 1.5. 導入時の注意点

- テナント・アプリケーションごとにコールバックURLやログアウトURLをホワイトリスト登録しておく必要がある。
- 無料プランはMAU（月間アクティブユーザー）数に上限があるため、成長に応じたプラン設計を事前に検討する。
