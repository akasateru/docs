## 1. ローカルLLM

- ChatGPTやGeminiのようなクラウドサービスを使わず、自分のPCや自社サーバー内のリソースだけで巨大言語モデル（LLM）を動かす仕組み
- ローカルLLMを動かす上で最も重要なのは、CPUではなく **「GPU（グラフィックボード）」とその「VRAM（ビデオメモリ）容量」**
  - VRAM 8GB以下: 7B（70億パラメータ）クラスの軽量モデルが限界。
  - VRAM 12GB ~ 16GB: 中規模モデル（14B前後）が快適に動作。
  - VRAM 24GB以上 (RTX 3090/4090): 70Bクラスの大型モデルも（工夫次第で）動作可能。
  - Macの場合: 「ユニファイドメモリ」を大量に積んだM1/M2/M3チップが非常に強力

## 2. Ollama

1. ダウンロード
<https://ollama.com/download>
2. 基本的なコマンド

| 操作           | コマンド               | 内容                                               |
| -------------- | ---------------------- | -------------------------------------------------- |
| 実行・チャット | ollama run <モデル名>  | モデルをダウンロードし、即座にチャットを開始する。 |
| モデル一覧     | ollama list            | PC内に保存されているモデルを表示する。             |
| モデル削除     | ollama rm <モデル名>   | 不要になったモデルを削除して容量を空ける。         |
| モデル取得のみ | ollama pull <モデル名> | チャットせずにダウンロードだけ行う。               |

## 3. ローカルLLMのメリット

- **プライバシー**: 外部サーバーにデータを送信せず、機密性の高いメモも安心して読み込ませることができる。
- **オフライン動作**: インターネット環境がなくても利用可能。
- **コスト**: 一度環境を構築すれば、API利用料（従量課金）を気にせず使い放題。

## 4. その他のツール

### 4.1. LM Studio (直感的なGUI)

モデルの検索、ダウンロード、チャットを一つのアプリで完結できる。

- **特徴**: GPUの設定などもGUIで細かく調整可能。Hugging Face上の多様なモデルを試せる。
- **導入**: [lmstudio.ai](https://lmstudio.ai/)

### 4.2. AnythingLLM (ドキュメント検索・RAGに最適)

`private-docs` フォルダを丸ごと読み込ませて、自分のメモに基づいた回答をさせるのに最も適している。

- **特徴**: フォルダを指定するだけで、ドキュメントのベクトル化（RAG）を自動で行ってくれる。
- **導入**: [useanything.com](https://useanything.com/)

#### 4.2.1. private-docs を読み込ませる手順

1. **AnythingLLMをインストール**: 上記公式サイトからダウンロードして起動。
2. **ワークスペースの作成**: `private-docs` という名前でワークスペースを作成。
3. **ドキュメントのインポート**:
   - `private-docs` フォルダをドラッグ＆ドキュメントエリアにドロップ。
   - 「Move to Workspace」→「Save and Embed」をクリック。
4. **チャット開始**: 自分のメモの内容に基づいたアドバイスをLLMから受けられる。

## 5. 推奨モデル (日本語性能重視)

- **Llama-3-8B-Instruct-v0.1-Japanese**: 日本語の理解力が高い。
- **Command R**: 文脈理解が深く、長いドキュメントの要約に適している。

## 6. 運用ルール

- **機密情報の扱い**: ローカル環境であっても、モデル自体に学習させる設定（一部のクラウド連携機能など）はオフにすること。
- **定期的なインデックス更新**: 新しいメモを追加した際は、AnythingLLM側で「Save and Embed」を再実行して情報を同期する。
