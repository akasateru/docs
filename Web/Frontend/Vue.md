## 1. Vue.js

- ユーザーインターフェース（UI）を構築するための、非常に人気のあるオープンソースのJavaScriptフレームワーク。
- 学習コストが低く、シンプルでありながら大規模な開発にも耐えうる柔軟性を備えているのが最大の特徴。

### 1.1. Vue.js の核となる概念

Vueを理解する上で欠かせない「3つの柱」がある。

- **リアクティブ・システム (Reactivity):** JavaScriptのデータ（状態）を変更すると、画面上の表示が**自動的に更新**される仕組み。面倒なDOM操作を手動で行う必要がない。
- **コンポーネント指向 (Component-based):** UIを「ボタン」「ヘッダー」「リスト」といった小さな部品（コンポーネント）に分割して開発。これにより、コードの再利用性やメンテナンス性が飛躍的に向上する。
- **宣言的レンダリング:** 「どうやって画面を書き換えるか」ではなく、「データがこうなったら、画面はこの状態になる」という結果を記述するスタイル。

### 1.2. Vue の主な特徴とメリット

#### 1.2.1. 学習のしやすさ

HTML、CSS、JavaScriptの基礎知識があれば、すぐに使い始めることができる。公式ドキュメントが非常に充実しており、日本語訳も高品質。

#### 1.2.2. 単一ファイルコンポーネント (SFC)

`.vue` という拡張子のファイルに、**HTML (template)**、**JavaScript (script)**、**CSS (style)** を1つにまとめて記述できる。関連するコードが1箇所に集まるため、非常に見通しが良い。

#### 1.2.3. 柔軟なスケーラビリティ

小さなウィジェット（既存ページの一部に導入）から、大規模なシングルページアプリケーション（SPA）まで、プロジェクトの規模に合わせて段階的に導入できる。

### 1.3. Options API vs Composition API

Vue 3からは、コードの書き方が大きく2種類に分かれた。

| 特徴           | Options API                                      | Composition API (Vue 3〜)                      |
| -------------- | ------------------------------------------------ | ---------------------------------------------- |
| **書き方**     | `data`, `methods`, `computed` など役割ごとに記述 | `setup` 内で機能（ロジック）ごとにまとめて記述 |
| **適した規模** | 小〜中規模、初心者向け                           | 大規模、ロジックの再利用を重視する場合         |
| **直感性**     | どこに何を書くか決まっていて分かりやすい         | 自由度が高く、JavaScriptに近い感覚で書ける     |

### 1.4. エコシステム

Vue単体だけでなく、周辺ツールも強力である。

- **Vite:** 超高速なビルドツール。
- **Vue Router:** ページ遷移（ルーティング）を管理するための公式ライブラリ。
- **Pinia:** アプリ全体のデータ（状態）を管理するためのストア。
- **Nuxt.js:** Vueをベースにしたフルスタックフレームワーク。SSR（サーバーサイドレンダリング）などを容易に実現できる。

## 2. Vueの機能

### 2.1. `<script setup>`

現在のVue 3開発において、最も標準的で推奨されている書き方が **`<script setup>`** である。Composition APIをより簡潔に書くための糖衣構文（シンタックスシュガー）で、コード量が大幅に削減される。

### 2.2. コンポーネント間の通信 (Props & Emits)

コンポーネントを分割した際、データを受け渡す標準的なルールが「Props down, Events up」である。

- **Props:** 親から子へデータを渡す（読み取り専用）。
- **Emits:** 子から親へイベントを通知する（アクションを起こす）。

これに加えて、深い階層のコンポーネントに直接データを渡したい場合は **`provide` / `inject`** という仕組みも用意されている。

### 2.3. ライフサイクル・フック

コンポーネントが生まれてから破棄されるまでの各タイミングで、特定の処理を実行できる。Vue 3のComposition APIでは、`onMounted` などがよく使われる。

- **onMounted:** 画面に表示された直後（APIのデータ取得など）。
- **onUpdated:** データが更新されて画面が再描画された後。
- **onUnmounted:** コンポーネントが消える直前（タイマーの解除など）。

### 2.4. パフォーマンス最適化のテクニック

Vueは標準でも高速だが、大規模アプリでは以下の手法が重要になる。

1. **v-show vs v-if:**
    - 頻繁に表示/非表示を切り替えるなら `v-show`（CSSのdisplayで制御）。
    - 切り替えが少ないなら `v-if`（DOMそのものを削除・生成）。
2. **shallowRef:**
    - 巨大なデータ（地図データや統計データなど）を扱う際、中身を深く監視しない `shallowRef` を使うことで、メモリ消費を抑えることができる。
3. **非同期コンポーネント:**
    - `defineAsyncComponent` を使うことで、そのコンポーネントが必要になったタイミングで初めてファイルを読み込む「遅延ロード」が可能になる。

### 2.5. イベントハンドラ (`v-on` / `@`)

ユーザーの操作（クリック、入力など）をきっかけにJavaScriptを実行する仕組み。

- **基本:** `@click="handlerName"` のように記述する。
- **インライン実行:** `@click="count++"` のように直接式を書くことも可能。
- **イベント修飾子:** Vue独自の便利な機能で、JavaScript側で `e.preventDefault()` などを書かずに済む。
  - `.stop`: イベント伝播を止める（stopPropagation）。
  - `.prevent`: 既定の動作をキャンセルする（preventDefault）。
  - `.once`: 1回だけ実行する。

### 2.6. データの監視 (`watch` / `watchEffect`)

データの変化をトリガーにして、副作用（APIを叩く、ログを出す、LocalStorageに保存するなど）を実行する。

| 機能              | 特徴                                                         | 主な用途                                                    |
| ----------------- | ------------------------------------------------------------ | ----------------------------------------------------------- |
| **`watch`**       | 特定のデータが「変化したとき」に実行。新旧の値を受け取れる。 | 値が変わったら特定のAPIを再実行する、といった明示的な処理。 |
| **`watchEffect`** | 依存しているデータを自動判別して、最初から即実行する。       | 複数の変数に依存する処理を簡潔に書きたいとき。              |

### 2.7. 非同期制御 (`Suspense`)

Vue 3で導入された、非同期処理などを待っている間にローディング状態をレンダリングするための特別なコンポーネント。

- **仕組み:** コンポーネント内の `setup` が `async`（Promiseを返す）の場合、その読み込みが終わるまで「待機中」の表示を出すことができる。
- **スロット:**
  - `#default`: 読み込みが終わった後に表示する内容。
  - `#fallback`: 読み込み中に表示する内容（ローディングスピナーなど）。

```vue
<Suspense>
  <template #default>
    <AsyncUserComponent />
  </template>
  <template #fallback>
    Loading...
  </template>
</Suspense>
```

### 2.8. 計算プロパティ (`computed`)

「データから派生する別の値」を効率的に作る機能である。

- **キャッシュ機能:** 依存しているデータが変わらない限り、何度呼び出しても計算をやり直さず、前回の結果を使い回す。
- **使い分け:**
  - **methods:** 呼び出されるたびに再計算。
  - **computed:** データが変わった時だけ再計算。

### 2.9. テンプレート参照 (`ref` ハンドラ)

DOM要素や子コンポーネントのインスタンスに直接アクセスしたい場合に使用する。

- `<input ref="inputRef">` のようにタグに `ref` 属性を付ける。
- スクリプト側で `const inputRef = ref(null)` と定義すると、マウント後にそのDOM要素を直接操作（`inputRef.value.focus()` など）できるようになる。
