# 1. shadcn/ui

## 1.1. 概要

shadcn/uiは、Radix UIとTailwind CSSをベースにしたReact用のコンポーネント集。npmパッケージとして依存追加するのではなく、CLIでコンポーネントのソースコードそのものをプロジェクトにコピーする方式が特徴。

## 1.2. 他のUIライブラリとの違い

| 項目           | shadcn/ui                         | MUI / Chakra UIなど          |
| -------------- | ----------------------------------- | ------------------------------ |
| 導入方法       | CLIでソースコードをコピー           | npmパッケージとしてインストール |
| カスタマイズ性 | コード自体を自由に編集できる        | props/themeでの上書きが中心    |
| バンドルサイズ | 使った分だけ（不要な機能は削れる）  | ライブラリ全体を含む傾向        |

## 1.3. セットアップの流れ

1. `npx shadcn@latest init` でTailwind設定やパスエイリアスを構成する。
2. `npx shadcn@latest add button` のようにコンポーネント単位で追加する。
3. `components/ui/` 配下にコピーされたソースを直接編集してカスタマイズする。

## 1.4. 特徴的な設計

- **Radix UI**: アクセシビリティ（キーボード操作、ARIA属性）が考慮されたヘッドレスコンポーネントを内部で使用。
- **class-variance-authority (cva)**: バリアント（サイズ、色などの見た目パターン）をタイプセーフに管理。
- **Tailwind CSS**: スタイリングはユーティリティクラスで行うため、デザインシステムのトークンと親和性が高い。

## 1.5. 向いているケース

- Next.jsなどでTailwindを既に採用しているプロジェクト。
- デザインを細部までカスタマイズしたいが、アクセシビリティの実装コストは避けたい場合。
