# Vue.js

## 1. 概要

- UIを構築するためのオープンソースのJavaScriptフレームワーク。
- 学習コストが低く、シンプルでありながら大規模な開発にも耐えうる柔軟性を備えているのが最大の特徴。

### 1.1. 主な特徴

- **リアクティブ・システム:** JavaScriptのデータを変更すると画面が**自動的に更新**される。手動のDOM操作が不要。
- **コンポーネント指向:** UIを小さな部品に分割して開発。コードの再利用性・メンテナンス性が向上。
- **宣言的レンダリング:** 「どうやって書き換えるか」ではなく「データがこうなったら画面はこの状態になる」という結果を記述するスタイル。
- **単一ファイルコンポーネント (SFC):** `.vue` ファイルに HTML / JavaScript / CSS をひとつにまとめて記述できる。

### 1.2. APIスタイル

Vue 3 から、コードの書き方が大きく2種類に分かれた。

| 特徴           | Options API                                      | Composition API (Vue 3〜)                      |
| -------------- | ------------------------------------------------ | ---------------------------------------------- |
| **書き方**     | `data`, `methods`, `computed` など役割ごとに記述 | `setup` 内で機能（ロジック）ごとにまとめて記述 |
| **適した規模** | 小〜中規模、初心者向け                           | 大規模、ロジックの再利用を重視する場合         |
| **直感性**     | どこに何を書くか決まっていて分かりやすい         | 自由度が高く、JavaScriptに近い感覚で書ける     |

現在の Vue 3 開発では **`<script setup>`** が最も標準的で推奨されている書き方。Composition API の糖衣構文で、コード量を大幅に削減できる。

### 1.3. テンプレート構文

#### 1.3.1. イベントハンドラ (`v-on` / `@`)

ユーザーの操作（クリック、入力など）をきっかけに JavaScript を実行する。

- **基本:** `@click="handlerName"`
- **インライン実行:** `@click="count++"` のように直接式を書くことも可能。
- **イベント修飾子:** `e.preventDefault()` などを JS 側に書かずに済む。
  - `.stop`: イベント伝播を止める（stopPropagation）
  - `.prevent`: 既定の動作をキャンセルする（preventDefault）
  - `.once`: 1回だけ実行する

#### 1.3.2. 条件付きレンダリング (`v-if` / `v-show`)

|                    | `v-if`               | `v-show`                          |
| ------------------ | -------------------- | --------------------------------- |
| **仕組み**         | DOM を削除・生成する | CSS の `display` で制御する       |
| **向いている場面** | 切り替えが少ない場合 | 頻繁に表示/非表示を切り替える場合 |

### 1.4. リアクティビティ

#### 1.4.1. 計算プロパティ (`computed`)

データから派生する値を効率的に作る機能。依存データが変わらない限り再計算せず、前回の結果をキャッシュする。

- `methods`: 呼び出されるたびに再計算
- `computed`: データが変わった時だけ再計算

#### 1.4.2. データの監視 (`watch` / `watchEffect`)

データの変化をトリガーにして副作用（API 呼び出し、LocalStorage への保存など）を実行する。

| 機能              | 特徴                                                     | 主な用途                                                    |
| ----------------- | -------------------------------------------------------- | ----------------------------------------------------------- |
| **`watch`**       | 特定のデータが変化したときに実行。新旧の値を受け取れる。 | 値が変わったら特定の API を再実行する、といった明示的な処理 |
| **`watchEffect`** | 依存しているデータを自動判別して即実行する。             | 複数の変数に依存する処理を簡潔に書きたいとき                |

#### 1.4.3. テンプレート参照 (`ref` ハンドラ)

DOM 要素や子コンポーネントのインスタンスに直接アクセスしたい場合に使用する。

- `<input ref="inputRef">` のようにタグに `ref` 属性を付ける。
- スクリプト側で `const inputRef = ref(null)` と定義すると、マウント後に `inputRef.value.focus()` などの直接操作が可能。

### 1.5. コンポーネント間通信

**Props down, Events up** が基本ルール。

- **Props:** 親から子へデータを渡す（読み取り専用）。
- **Emits:** 子から親へイベントを通知する。

深い階層のコンポーネントに直接データを渡したい場合は **`provide` / `inject`** を使う。

### 1.6. ライフサイクル

コンポーネントが生まれてから破棄されるまでの各タイミングで特定の処理を実行できる。

| フック        | タイミング                         | 用途例           |
| ------------- | ---------------------------------- | ---------------- |
| `onMounted`   | 画面に表示された直後               | API のデータ取得 |
| `onUpdated`   | データ更新後に画面が再描画された後 | -                |
| `onUnmounted` | コンポーネントが消える直前         | タイマーの解除   |

### 1.7. 非同期・パフォーマンス

#### 1.7.1. 非同期制御 (`Suspense`)

非同期処理の待機中にローディング状態をレンダリングするための特別なコンポーネント（Vue 3〜）。
コンポーネントの `setup` が `async` の場合、読み込みが終わるまで fallback を表示できる。

```vue
<Suspense>
  <template #default>
    <AsyncUserComponent />
  </template>
  <template #fallback>
    Loading...
  </template>
</Suspense>
```

#### 1.7.2. 非同期コンポーネント

`defineAsyncComponent` を使うことで、コンポーネントが必要になったタイミングで初めてファイルを読み込む「遅延ロード」が可能。

#### 1.7.3. `shallowRef`

巨大なデータ（地図データや統計データなど）を扱う際、中身を深く監視しない `shallowRef` を使うとメモリ消費を抑えられる。
