# 1. SendGrid

## 1.1. 概要

SendGridはTwilio傘下のメール配信サービス（MaaS）。トランザクションメール（パスワードリセット、通知など）とマーケティングメール（キャンペーン配信）の両方に対応する。

## 1.2. 主な機能

- **Mail Send API**: アプリケーションからHTTP経由でメールを送信するAPI。
- **Dynamic Templates**: Handlebars構文でメール本文をテンプレート化し、変数を差し込める。
- **Webhook（Event Webhook）**: 配信結果（delivered, bounce, open, clickなど）をリアルタイムで受信できる。
- **Suppression Management**: バウンスやスパム報告があったアドレスへの再送信を自動的に抑制する。

## 1.3. 送信の到達率を上げるための設定

| 項目             | 内容                                                     |
| ---------------- | ---------------------------------------------------------- |
| SPF              | 送信元ドメインがそのメールサーバーからの送信を許可していることを示すDNSレコード |
| DKIM             | メールに電子署名を付与し、改ざんされていないことを証明     |
| DMARC            | SPF/DKIMの認証結果に基づき、なりすましメールの扱いを指定    |
| ドメイン認証     | SendGridの管理画面から送信ドメインを認証しておくと到達率が向上 |

## 1.4. 実装の流れ（Node.jsの例）

1. APIキーを発行し、環境変数に設定する。
2. `@sendgrid/mail` パッケージをインストールする。
3. `msg` オブジェクトに `to`, `from`, `subject`, テンプレートIDなどを指定して `send()` を呼び出す。
4. Webhookエンドポイントを用意し、配信結果をログに記録する。

## 1.5. 注意点

- `from` に指定するドメインは事前にドメイン認証しておかないと、迷惑メール判定されやすい。
- 大量送信時はレート制限に注意し、バッチ送信APIやキューイングを検討する。
