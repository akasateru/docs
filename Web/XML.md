# XML

XML（eXtensible Markup Language）は、データを構造化して記述するためのマークアップ言語の仕様（メタ言語）。タグの意味・語彙を自分で自由に定義できるが、文書として満たすべき構文規則は厳格に決まっている。

## 1. 概要

```xml
<book>
  <title>吾輩は猫である</title>
  <author>夏目漱石</author>
</book>
```

`<book>`や`<title>`はXML自体が決めた語彙ではなく、利用者が定義したもの。用途はデータの受け渡し・保存（設定ファイル、RSS、SOAP APIなど）が中心。

## 2. XMLの構文規則（Well-formedness）

タグの名前が何であれ、XMLとして正しい（well-formed）文書は以下を必ず満たす。

1. ルート要素は1つだけ
2. 開いたタグは必ず閉じる（`<br></br>`、自己終了なら`<br/>`）
3. タグは正しく入れ子になる（`<a><b></a></b>`はNG）
4. 属性値は必ずクォートで囲む（`class="box"`）
5. タグ名の大文字小文字を区別する
6. `<`や`&`などの特殊文字はエスケープする

これらを満たしていれば、タグの語彙が独自でもXMLパーサーで正しくパースできる。

## 3. HTMLとの関係

**「HTMLはXMLに内包される」は誤り。** タグの語彙が自由に作れることと、ある文書がXMLの集合に含まれるかどうかは別の軸。含まれるかどうかを決めるのは「2. XMLの構文規則」を満たしているかどうかであり、語彙の自由度ではない。

HTML5は独自のタグ語彙（`<h1>`, `<div>`など）を持つが、構文規則はXMLより緩く、別物になっている。

| ルール | XML | HTML5 |
|---|---|---|
| 閉じタグ必須 | 必須（`<br></br>`） | 省略可（`<br>`でOK） |
| 属性値のクォート | 必須 | 省略可（`class=box`もOK） |
| 大文字小文字 | 区別する | 区別しない |
| 未定義タグ | エラー | ブラウザが黙って解釈 |

```html
<br>                      <!-- 閉じタグなしOK（XMLならエラー） -->
<input type=text>         <!-- クォートなしOK（XMLならエラー） -->
<li>りんご
<li>みかん               <!-- 閉じタグ省略OK（XMLならエラー） -->
<DIV>text</div>           <!-- 大文字小文字混在OK（XMLならエラー） -->
```

これらはすべて有効なHTML5だが、XMLパーサーに読ませるとエラーになる。つまり一般的なHTML5文書はXMLの構文規則を満たしておらず、XMLの集合には含まれない。

```text
XML（構文規則の集合）
 └─ この規則を満たす文書だけがXML
     ├─ RSS、SVG、XHTML ...（規則を満たす）
     └─ 一般的なHTML5文書 ...（規則を満たさない＝XMLの外）
```

### 3.1. 補足: XHTML

過去に「HTMLの語彙をXMLの厳格な構文規則で書く」**XHTML**という規格が存在した（閉じタグ必須、属性クォート必須など）。これは文字通り「XMLで書かれたHTML」であり、XMLの集合に含まれる数少ないHTML系フォーマットだった。しかし現在主流のHTML5はXHTMLの後継ではなく、独自の緩い構文規則（HTML Living Standardの独自パーサ）を採用する別の道を選んでいる。
