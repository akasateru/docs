# k6

Grafana が提供する負荷テストツール。テストスクリプトを JavaScript / TypeScript で記述する。

---

## インストール

```bash
# macOS
brew install k6

# その他: https://grafana.com/docs/k6/latest/set-up/install-k6/
```

---

## 基本的な使い方

```bash
k6 new            # テンプレートを生成
k6 run script.js  # テスト実行

# 環境変数を渡す
k6 run -e TARGET_URL=https://example.com script.js

# CLIでVU数・実行時間を上書き
k6 run --vus 10 --duration 30s script.js
```

## スクリプト構成

```js
import http from 'k6/http';
import { check, sleep } from 'k6';

// オプション設定
export const options = {
  stages: [
    { duration: '30s', target: 20 }, // 30秒かけて20VUまで増やす
    { duration: '1m',  target: 20 }, // 1分間20VUを維持
    { duration: '20s', target: 0  }, // 20秒で0VUへ戻す
  ],
  thresholds: {
    'http_req_failed':   ['rate<0.01'],   // エラー率 1% 未満
    'http_req_duration': ['p(95)<500'],   // 95%ile が 500ms 以内
  },
};

// メインのVUコード（各VUがループ実行）
export default function () {
  const url = __ENV.TARGET_URL || 'https://test-api.k6.io/public/crocodiles/';

  const res = http.get(url, {
    headers: { 'Content-Type': 'application/json' },
  });

  check(res, {
    'status is 200':      (r) => r.status === 200,
    'protocol is HTTP/2': (r) => r.proto === 'HTTP/2.0' || r.proto === 'h2',
  });

  sleep(1); // ユーザーの行動間隔をシミュレート
}
```

## テストライフサイクル

```js
// 1. init: ファイル読み込み時に1回だけ実行（VUごとではない）
import { SharedArray } from 'k6/data';
const data = new SharedArray('users', () => JSON.parse(open('./users.json')));

// 2. setup: VU開始前に1回実行。戻り値をVUコードとteardownに渡せる
export function setup() {
  return { token: 'abc123' };
}

// 3. VU Code (default): 各VUが繰り返し実行
export default function (data) {
  // data.token が使える
}

// 4. teardown: テスト終了後に1回実行
export function teardown(data) {
  // クリーンアップ処理
}
```

| ステージ | 実行タイミング | 主な用途 |
| --- | --- | --- |
| init | 起動時・VUごとに1回 | import、ファイル読み込み |
| setup | VUコード開始前に1回 | 認証、テストデータ準備 |
| VU Code | 各VUがループ実行 | HTTPリクエスト、チェック |
| teardown | 終了後に1回 | リソース解放、後処理 |

## 実行オプション

| オプション | スクリプト変数 | 説明 |
| --- | --- | --- |
| `--vus` | `vus` | 仮想ユーザー数 |
| `--duration` | `duration` | 実行時間（例: `30s`, `1m`） |
| `--iterations` | `iterations` | 合計実行回数（達したら終了） |
| `--rps` | `rps` | 秒間リクエスト数の上限 |
| `--batch` | `batch` | `http.batch()` の最大並列接続数 |
| `--no-connection-reuse` | `noConnectionReuse` | リクエストごとにTCP接続を張り直す |
| `--user-agent` | `userAgent` | User-Agent を変更 |
| `--out` | CLIのみ | 出力先（`json`, `influxdb`, `csv` など） |

## シナリオと Executor

`options.scenarios` で複数シナリオを並行実行できる。

```js
export const options = {
  scenarios: {
    smoke: {
      executor: 'constant-vus',
      vus: 1,
      duration: '30s',
    },
    load: {
      executor: 'ramping-arrival-rate',
      startTime: '30s',
      preAllocatedVUs: 50,
      stages: [
        { target: 100, duration: '1m' },
        { target: 0,   duration: '30s' },
      ],
    },
  },
};
```

### Executor の種類

| Executor | 制御軸 | 特徴 |
| --- | --- | --- |
| `constant-vus` | VU数 | 固定VU数で一定時間実行 |
| `ramping-vus` | VU数 | VU数を段階的に増減（`stages` と同等） |
| `constant-arrival-rate` | RPS | 秒間リクエスト数を一定に保つ |
| `ramping-arrival-rate` | RPS | 秒間リクエスト数を段階的に変化 |
| `per-vu-iterations` | 反復回数 | 各VUが指定回数だけ実行 |
| `shared-iterations` | 反復回数 | 全VUで合計回数をシェア |
| `externally-controlled` | 外部制御 | REST API / CLI でリアルタイム制御 |

### シナリオ共通パラメータ

| パラメータ | 型 | 説明 |
| --- | --- | --- |
| `executor` | string | 必須。上表から選択 |
| `exec` | string | 実行する関数名（デフォルト: `default`） |
| `startTime` | string | テスト開始からこのシナリオが起動するまでの待機時間 |
| `gracefulStop` | string | 終了後にイテレーション完了を待つ時間（デフォルト: `30s`） |
| `env` | object | シナリオ内だけで有効な環境変数 |
| `tags` | object | メトリクスに付与するカスタムタグ |

## 主要メトリクス

| メトリクス | 説明 |
| --- | --- |
| `http_reqs` | 総リクエスト数 |
| `http_req_duration` | リクエスト完了までの合計時間 |
| `http_req_failed` | 失敗リクエストの割合 |
| `http_req_blocked` | リクエスト開始前のブロック時間 |
| `http_req_connecting` | TCP接続確立にかかった時間 |
| `checks` | `check()` の成功割合 |
| `iteration_duration` | 1イテレーションの所要時間 |
| `vus` | アクティブなVU数 |
| `vus_max` | テスト中の最大VU数 |
