## 1. k6

- 負荷テストツール
- JavaScriptで記述する

## 2. 使い方

### 2.1. インストール

<https://grafana.com/docs/k6/latest/set-up/install-k6/>

### 2.2. テスト実行方法

```bash
## テストファイルを作成
k6 new

## テスト実行
k6 run script.js
```

### 2.3. 実装例

1. オプションを設定
   1. 成功基準も設定
2. テスト実行
3. レスポンスの検証

```js
import http from 'k6/http';
import { check, sleep } from 'k6';

// 1. オプション設定（テストの構成）
export const options = {
  stages: [
    { duration: '30s', target: 20 }, // 30秒かけて20ユーザーまで増やす
    { duration: '1m',  target: 20 }, // 1分間20ユーザーを維持
    { duration: '20s', target: 0  }, // 20秒で0ユーザーへ戻す
  ],
  thresholds: {
    // 成功基準の設定
    'http_req_failed': ['rate<0.01'],   // エラー率は1%未満であること
    'http_req_duration': ['p(95)<500'], // 95%のリクエストが500ms以内であること
  },
};

// 2. テストの実行内容
export default function () {
  // 環境変数からURLを取得（デフォルトは example.com）
  const url = __ENV.TARGET_URL || 'https://test-api.k6.io/public/crocodiles/';

  const params = {
    headers: {
      'Content-Type': 'application/json',
    },
  };

  // GETリクエストの送信
  const res = http.get(url, params);

  // レスポンスの検証（Check）
  check(res, {
    'status is 200': (r) => r.status === 200,
    'protocol is HTTP/2': (r) => r.proto === 'HTTP/2.0' || r.proto === 'h2',
  });

  // ユーザーの行動間隔をシミュレート（1秒待機）
  sleep(1);
}
```

## 3. 実行オプション

| オプション名          | スクリプト内変数  | 内容                                                                                    |
| --------------------- | ----------------- | --------------------------------------------------------------------------------------- |
| --vus                 | vus               | 仮想ユーザー数 (Virtual Users)。同時に実行する人数。                                    |
| --duration            | duration          | 実行時間。例: 30s, 1m。                                                                 |
| --iterations          | iterations        | 合計実行回数。指定した回数に達したら終了する。                                          |
| --rps                 | rps               | 1秒あたりのリクエスト数制限。負荷を一定に保つ際に使用。                                 |
| --out                 | (CLIのみ)         | 出力先。json, influxdb, csv など外部へデータを送る。                                    |
| --rps                 | rps               | スループット固定。VUs数に関わらず、秒間のリクエスト数を一定に保ちたい時に使用。         |
| --batch               | batch             | 並列リクエスト数。http.batch() で同時に送る最大接続数を制限する。                       |
| --no-connection-reuse | noConnectionReuse | 接続の再利用禁止。リクエストごとにTCP接続を貼り直す（新規訪問ユーザーのシミュレート）。 |
| --user-agent          | userAgent         | User-Agentの変更。特定のブラウザやデバイスを装う。                                      |

## 4. シナリオ

どの `executor` を選んでも `scenarios` オブジェクト内で設定できる共通項目。

| 引数名             | 型     | 説明                                                                                  |
| ------------------ | ------ | ------------------------------------------------------------------------------------- |
| **`executor`**     | string | **必須。** 実行モデル（例: `constant-vus`, `ramping-arrival-rate` など）を指定。      |
| **`exec`**         | string | 実行するJS関数名（デフォルトは `default`）。シナリオごとに別の関数を呼べます。        |
| **`startTime`**    | string | テスト開始からこのシナリオが動き出すまでの待ち時間（例: `"30s"`）。                   |
| **`gracefulStop`** | string | テスト終了後、実行中の反復（Iteration）が完了するまで待つ時間（デフォルト `"30s"`）。 |
| **`env`**          | object | そのシナリオ内だけで有効な環境変数。                                                  |
| **`tags`**         | object | メトリクスに付与する独自のタグ。                                                      |

## 5. メトリクスの見方

| 指標名              | 意味                                          |
| ------------------- | --------------------------------------------- |
| checks              | チェック関数の成功割合                        |
| http_req_duration   | HTTPリクエストの完了にかかった合計時間        |
| http_req_failed     | 失敗したHTTPリクエストの割合                  |
| iteration_duration  | 1回のイテレーションを完了するのにかかった時間 |
| vus                 | アクティブな仮想ユーザー(VU)の数              |
| vus_max             | 最大VU数                                      |
| http_reqs           | テスト中に生成されたHTTPリクエストの総数      |
| http_req_blocked    | リクエスト開始前にブロックされている時間      |
| http_req_connecting | TCP接続の確立にかかった時間                   |

## 6. テストライフサイクル

| ステージ | 目的                                                                                                                                                           |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| init     | テストの開始前に一度だけ実行されるステージ。グローバル変数の宣言やテスト全体で使用する設定の初期化などを行う。                                                 |
| setup    | テストのdefault関数(VU code)が実行される前に一度だけ実行されるステージ。テストデータの準備や外部からのデータの取り込みなど、テスト実行に必要な前処理を行いう。 |
| VU Code  | 各VU(仮想ユーザー)によって実行されるメインのテストスクリプト。HTTPリクエストの送信やカスタムメトリクスの計測など、テストの核となるアクションが含まれる。       |
| teardown | テストのメイン部分の実行が終了した後に一度だけ実行されるステージ。テストで使用したリソースのクリーンアップや結果の後処理などを行う。                           |
