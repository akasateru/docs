# 1. PagerDuty

## 1.1. 概要

PagerDutyは障害・アラートの通知とオンコール対応を管理するインシデント管理プラットフォーム。監視ツール（Datadog, CloudWatchなど）からのアラートを受け取り、担当者へ電話・SMS・Push通知でエスカレーションする。

## 1.2. 主要な概念

| 概念               | 内容                                                     |
| -------------------- | ----------------------------------------------------------- |
| Service              | 監視対象のシステム単位。アラートの受け口となる               |
| Incident              | アラートから作成されるインシデント。対応状況を追跡する単位   |
| Escalation Policy     | 誰にどの順番・間隔で通知するかのルール                       |
| On-call Schedule      | オンコール担当者のローテーション設定                         |

## 1.3. エスカレーションの流れ

1. 監視ツールからアラートが発報され、Incidentが作成される。
2. Escalation Policyに従って一次担当者に通知が送られる。
3. 一定時間（例: 5分）以内にAcknowledge（対応中の意思表示）がなければ、次の担当者にエスカレーションされる。
4. 対応完了後、Incidentをresolveしてクローズする。

## 1.4. 運用上のポイント

- **アラート疲れ（Alert Fatigue）の防止**: 本当に対応が必要な重要度の高いアラートのみが通知されるよう、閾値やルールを適切にチューニングする。
- **Runbook（対応手順書）のリンク**: Incident発生時に、対応者が迷わないようRunbookをIncidentに紐付けておく。
- **ポストモーテムとの連携**: Incidentのタイムラインをそのまま振り返り（ポストモーテム）の資料として活用できる。
