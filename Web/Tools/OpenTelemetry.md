# 1. OpenTelemetry

## 1.1. 概要

OpenTelemetryは、分散トレーシング・メトリクス・ログを収集するためのベンダーニュートラルな観測（Observability）フレームワーク。CNCFのプロジェクトで、特定のAPM製品にロックインされずに計装できるのが特徴。

## 1.2. 主要コンポーネント

| コンポーネント | 役割 |
| ------------------- | ------------------------------------------------------------ |
| API / SDK | アプリケーションコードに計装を組み込むためのライブラリ群 |
| Collector | 各アプリから送られたテレメトリデータを受信・加工・転送するプロキシ |
| Exporter | Collectorや各種バックエンド（Datadog, New Relic, Jaegerなど）へデータを送る出力先 |

## 1.3. 計装（Instrumentation）の種類

- **自動計装（Auto Instrumentation）**: 主要フレームワーク（Express, Next.jsなど）に対して、コード変更を最小限にトレースを仕込める。
- **手動計装（Manual Instrumentation）**: 独自のスパンやカスタム属性を明示的にコードへ追加する。

## 1.4. Collectorの役割

- 複数サービスからのテレメトリを一箇所に集約し、サンプリングやバッチ処理を行う。
- バックエンドを切り替える際、アプリ側のコードを変更せずにCollectorの出力設定だけ変えれば良い。

## 1.5. トレースの基本用語

- **Trace**: 1つのリクエストが複数サービスを通過する一連の流れ全体。
- **Span**: Trace内の個々の処理単位（例: DBクエリ、外部API呼び出し）。
- **Context Propagation**: サービス間をまたいでTrace IDを伝播させる仕組み。
