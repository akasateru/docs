# 1. Google Analytics（GA4）

## 1.1. 概要

Google Analytics 4（GA4）はWeb・アプリのアクセス解析ツール。旧UA（ユニバーサルアナリティクス）とは計測モデルが根本的に異なり、全てのユーザー行動を「イベント」として一元的に記録する設計になっている。

## 1.2. UAとの主な違い

| 項目           | UA（旧）                     | GA4                              |
| -------------- | -------------------------------- | ----------------------------------- |
| 計測モデル     | セッションベース                 | イベントベース                     |
| クロスプラットフォーム | Webのみ                    | Web・アプリを統合計測可能           |
| Cookie依存     | 高い                              | 機械学習による補完でCookie依存を低減 |

## 1.3. 主要な概念

- **イベント（Event）**: ページビュー、クリック、スクロールなど全ての計測単位。`page_view` や `purchase` などがある。
- **コンバージョン**: 重要なイベントをコンバージョンとしてマークし、目標達成として集計する。
- **オーディエンス**: 特定条件を満たすユーザー群をセグメントとして定義し、広告や分析に活用する。
- **探索レポート（Explore）**: 自由な軸でデータを掛け合わせて分析できるカスタムレポート機能。

## 1.4. 導入方法

- **gtag.js**: サイトに直接計測タグを埋め込む方式。
- **Google Tag Manager（GTM）**: タグ管理ツール経由で計測設定を一元管理する方式。コード変更なしにタグの追加・修正ができるため実務ではこちらが主流。

## 1.5. 運用時の注意点

- カスタムイベントの命名規則（snake_caseが推奨）を決めてチームで統一しておく。
- 個人情報（メールアドレスなど）をイベントパラメータに含めないよう注意する（Googleの利用規約違反になる）。
- BigQueryへのエクスポート機能を使うと、生データレベルでの詳細な分析やSQLでの集計が可能。
