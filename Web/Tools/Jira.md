## 1. Jira

Jiraのエピックとは、複数のタスクやストーリーを一定のジャンルでグループにしたものです。 作業を整理したり、階層を作成したり、開発の優先順位をつけるときに役立ちます。

Jiraでは、プロジェクト達成に必要な作業を分解し、構造化するために、「ストーリー」「エピック」「タスク」「イニシアチブ」の4つに分類することができます。
それぞれの意味、役割の違いは以下の通りです。

| イニシアチブ | 複数のエピックが組み合わさって、さらに大きな価値・目標になったもの             |
| ------------ | ------------------------------------------------------------------------------ |
| エピック     | エンドユーザーの要望に基づいて、複数のタスクやストーリーをグループ分けしたもの |
| ストーリー   | エンドユーザーに届けたい価値を機能・作業単位でまとめたもの                     |
| タスク       | 開発メンバーが行う個別の作業・ToDo項目                                         |

### 1.1. ストーリーとの違い

Jiraのストーリーは、複数のタスクを包括したもので、機能・作業単位でまとめられます。
一方で、Jiraのエピックは、複数のタスクやストーリーを包括し、ストーリーよりもより大きな単位でグループ分けされるのが特徴です。
ストーリーが機能・作業をあらわすのに対し、エピックは目的や概要をあらわします。関連するストーリーを完了させることで、エピックが完了します。

### 1.2. タスクとの違い

Jiraのタスクとは、ストーリーやエピックを完了するために必要な作業のことです。
ストーリーは、エンドユーザーの要望・視点に基づいてあらわしますが、タスクは作業をする開発メンバーの視点であらわすという違いがあります。

## 2. 1\. Jiraの標準的な階層構造

Jiraでは、上位から下位へ向かって以下のように紐付きます。

1. **イニシアチブ**（複数のエピックを束ねる）※Premiumプラン以上
2. **エピック**（大きな機能・目的）
3. **ストーリー / タスク / バグ**（標準の課題タイプ：これらは**同列**です）
4. **サブタスク**（ストーリーやタスクをさらに細分化した最小作業単位）

## 3. ここが混乱しやすいポイント

- **「ストーリー」と「タスク」は親子ではない：** Jiraのデフォルト設定では、ストーリーとタスクはどちらも「標準の課題レベル（Level 0）」に属する**兄弟**のような関係です。そのため、「ストーリーの中にタスクを入れる」ことはできません。
- **ストーリーの配下は「サブタスク」のみ：** ストーリー（またはタスク）をさらに細かく分けたい場合は、\*\*「サブタスク」\*\*という特殊な課題タイプを使用します。

* * *

## 4. 2\. なぜ「ストーリーの下にタスク」が置けないのか？

Jiraの思想（アジャイル・スクラム）に基づくと、以下のような使い分けを想定しているためです。

- **ストーリー：** ユーザーから見た価値（例：「ログインできる」）
- **サブタスク：** そのストーリーを実現するための具体的な作業（例：「DBのテーブル作成」「バリデーションの実装」）

もし「ストーリー（Level 0）」の中に「タスク（Level 0）」を入れようとすると、同じ階層同士で親子関係を作ることになり、管理が複雑化するため制限されています。

* * *

## 5. 3\. もしどうしても階層をカスタマイズしたい場合は？

もし「ストーリー ＞ タスク ＞ サブタスク」のような独自の3段（あるいはそれ以上）の階層を作りたい場合は、以下の方法があります。

## 6. A. 課題のリンク（Issue Link）を使う

親子関係ではなく、「関連している」「ブロックしている」といった**リンク機能**で繋ぎます。階層構造にはなりませんが、追跡は可能です。

## 7. B. 階層設定をカスタマイズする（Jira Premiumのみ）

Jira Premiumプラン以上の\*\*「高度なロードマップ（Advanced Roadmaps）」\*\*機能を使えば、管理設定から「イニシアチブ ＞ エピック ＞ ストーリー ＞ サブタスク」以外の独自の階層レベルを追加・変更することが可能です。
