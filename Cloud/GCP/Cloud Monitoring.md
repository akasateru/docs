# 1. Cloud Monitoring

## 1.1. 概要

Cloud MonitoringはGoogle Cloudのモニタリングサービス。GCPリソース（Compute Engine, Cloud Run, GKEなど）のメトリクスを自動収集し、ダッシュボード・アラート・アップタイムチェックなどを提供する。旧称はStackdriver Monitoring。

## 1.2. 主要な機能

| 機能 | 内容 |
| -------------------- | ----------------------------------------------------------- |
| メトリクス収集 | CPU使用率、リクエスト数、レイテンシなどを自動・カスタムで収集 |
| ダッシュボード | 収集したメトリクスをグラフとして可視化 |
| アラートポリシー | 閾値を超えた際に通知（メール、Slack、PagerDutyなど）を送る |
| アップタイムチェック | 外部からエンドポイントへ定期的にリクエストし死活監視する |
| SLOモニタリング | サービスレベル目標（SLO）の達成状況を追跡 |

## 1.3. メトリクスの種類

- **組み込みメトリクス**: 各GCPサービスが自動的に送信するメトリクス（例: Cloud Runのリクエスト数、レスポンスコード別カウント）。
- **カスタムメトリクス**: アプリケーション独自の指標をOpenTelemetryやClient Libraryで送信したもの。

## 1.4. アラートポリシーの基本構成

1. 監視対象のメトリクスと条件（例: エラー率が5分間で5%を超えたら）を設定する。
2. 通知先（Notification Channel）を設定する（メール、Slack、PagerDutyなど）。
3. インシデントが自動解消したかどうかも合わせて通知するかを設定する。

## 1.5. ログとの連携

- Cloud Loggingと連携し、ログベースの指標（Log-based Metrics）を作成してアラート条件に使うことができる。
- トレース（Cloud Trace）と合わせることで、メトリクスの異常からトレースへドリルダウンして原因調査ができる。
