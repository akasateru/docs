## 1. Cloud CDN

- Cloud Load Balancing（外部 HTTP(S) ロードバランサ）**のチェックボックスを一つオンにするだけで有効化できる

### 1.1. メリット

| メリット               | 内容                                                                                                      |
| ---------------------- | --------------------------------------------------------------------------------------------------------- |
| **ユーザー体験の向上** | レイテンシ（遅延）が減り、ページが瞬時に開くようになります。                                              |
| **オリジンの負荷軽減** | リクエストの多くを CDN が肩代わりするため、Cloud Run などのインスタンス数を抑えられ、コスト削減に繋がる。 |
| **耐障害性の向上**     | 急激なアクセス集中（スパイク）が発生しても、CDN が防波堤となりオリジンがダウンするのを防ぐ。              |

## 2. エンジニアがハマる「キャッシュ事故」

CDN は非常に便利ですが、設定ミスによる事故も多いです。

- **キャッシュパージ（消去）の遅れ:** コードを修正してデプロイしたのに、CDN が古いファイルを返し続けてユーザーに反映されない。
- **個人情報のキャッシュ:** 誤って「マイページ」などの個人情報を含む動的コンテンツをキャッシュしてしまい、他のユーザーに見えてしまう
