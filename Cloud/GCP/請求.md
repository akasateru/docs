## 1. 請求の階層構造

- 組織 (Organization): 会社全体。すべてのプロジェクトと請求がこの下に属す。
- 請求先アカウント (Billing Account): 「財布」の役割。クレジットカードや銀行振込の設定を行い、複数のプロジェクトを1つの財布に紐づける。
- プロジェクト (Project): 開発の単位。Gemini APIなどのリソースは必ず特定のプロジェクトに属し、そのプロジェクトが紐づいている「請求先アカウント」に課金される。
- お支払いプロファイル: Google全体の支払い情報。請求先アカウントのさらに外側にある。

## 2. 課金の仕組み

### 2.1. 従量課金制 (Pay-as-you-go)

- 基本は「秒単位」や「使用量（トークン数、データ量）」に応じた課金
- 初期費用や解約金はなし

### 2.2. サービスごとの課金軸

Gemini CLIなどで特に関係するのは以下の3点。

- コンピュート: 実行時間（秒単位）。
- ストレージ: 保存データ量（GB/月単位）。
- API / トークン: 処理した文字数やトークン数（Geminiなど）。

### 2.3. 割引制度

- 無料枠 (Always Free): 毎月一定量まで無料。
- 継続利用割引: 設定不要で、長時間動かし続けると自動で適用（GCEなど）。
- 確約利用割引 (CUD): 1年または3年の利用を約束することで、最大50〜70%割引。
